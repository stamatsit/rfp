# Premium Polish Vocabulary

**MANDATORY READ** for any agent touching client app polish.

This file defines the **only** visual changes any agent should make. Apply these patterns *consistently* across every component. The goal is unification — the app should feel like one premium product, not 21 patched pages.

## Hard rules — never violate

- **Visual only.** Never change component logic, hooks, state, props, exports, imports, file structure, or routing.
- **Never refactor.** Don't extract components, rename variables, reorder code, or "while I'm here" anything.
- **Never break dark mode.** Every change must work in both `:root` and `.dark`.
- **Never change layout dimensions.** No padding/margin changes that affect layout flow. Tweaks to gaps and inner padding are OK only when applying the spacing-rhythm rule below.
- **Don't add motion that loops.** No pulsing, breathing, floating ambient animations. Motion is reserved for state transitions only.
- **Don't add gradients to text** unless replacing an existing colored text in a hero/headline only. Body text and UI labels stay solid.
- **Don't add `backdrop-blur` to anything that scrolls** — perf cliff. Only on fixed/sticky positioned overlays (modals, sticky nav, command palette).
- **Always test changes work in the existing dark mode** — every Tailwind class with a color must have a `dark:` variant if the surrounding code does.

## The vocabulary — apply these patterns everywhere

### 1. Cards — hover lift
- Default state: existing
- Add: `transition-all duration-300 ease-[cubic-bezier(0.22,1,0.36,1)]`
- On hover: `hover:-translate-y-0.5 hover:shadow-lg`
- For interactive cards (clickable): also add `active:translate-y-0 active:shadow-md`
- Skip cards already inside a list/grid that has its own hover scheme — don't double up.

### 2. Buttons — micro-press + smooth color
All button variants (the `button.tsx` primitive already has good baseline). Audit every `<button>` not using the primitive and add:
- `transition-all duration-150 ease-out`
- `active:scale-[0.97]` (existing primitive uses `0.98` — keep that, don't change)
- Hover: brightness change OR background shift — don't add both
- Disabled: `disabled:opacity-50 disabled:cursor-not-allowed disabled:hover:translate-y-0` and remove all hover effects when disabled

### 3. Focus rings — keyboard accessibility AND polish
Every interactive element needs a visible keyboard focus ring distinct from hover:
- `focus-visible:outline-none focus-visible:ring-4 focus-visible:ring-blue-500/15 focus-visible:ring-offset-2 focus-visible:ring-offset-white dark:focus-visible:ring-offset-slate-950`
- For destructive: `ring-red-500/15`
- For success: `ring-emerald-500/15`
- Use `focus-visible:` not `focus:` — cleaner mouse experience.

### 4. Inputs — refined focus state
`Input`/`Textarea` primitives need:
- `transition-colors duration-150`
- `hover:border-slate-300 dark:hover:border-slate-600` (subtle)
- `focus-visible:border-blue-500 focus-visible:ring-4 focus-visible:ring-blue-500/10`
- Remove existing harsh `focus:border-2` if present — use `focus-visible:ring` instead

### 5. Modals & Dialogs — entrance animation
Replace instant appear with:
- Backdrop: `data-[state=open]:animate-in data-[state=open]:fade-in-0 data-[state=open]:duration-200`
- Backdrop: add `backdrop-blur-sm bg-black/40` (was probably `bg-black/50`)
- Content: `data-[state=open]:animate-in data-[state=open]:fade-in-0 data-[state=open]:zoom-in-95 data-[state=open]:slide-in-from-bottom-2 data-[state=open]:duration-250 data-[state=open]:ease-[cubic-bezier(0.22,1,0.36,1)]`
- Exit (Radix supports it): `data-[state=closed]:animate-out data-[state=closed]:fade-out-0 data-[state=closed]:zoom-out-95 data-[state=closed]:duration-200`

If using `tailwindcss-animate` (it's installed), the `animate-in`/`animate-out` utilities work out of the box.

### 6. Loading states — skeletons over spinners
For any loading region **larger than 200px in either dimension**, replace bare `<Loader2 />` spinners with the `skeleton` primitive. Keep spinners only for inline contexts (button loading state, small icons, inline progress).

When using skeletons, use the **shimmer animation** that already exists in `index.css`:
```html
<div className="shimmer h-4 w-32 rounded" />
```

### 7. Empty states — entrance + restraint
Wrap empty-state containers with:
- `animate-in fade-in-0 slide-in-from-bottom-2 duration-500`
- Icon at `text-slate-300 dark:text-slate-700` and size `48` (most are inconsistent)
- Heading: `text-base font-medium text-slate-700 dark:text-slate-300` (NOT bold)
- Description: `text-sm text-slate-500 dark:text-slate-400 max-w-sm mx-auto`
- Spacing: `space-y-3`

### 8. Lists & grids — stagger on mount only
For result lists/grids of < 30 items, stagger children. The `.stagger-children` class already exists in `index.css`. Apply to the wrapping container. Do NOT re-trigger on filter changes — only initial mount.

### 9. Tooltips, dropdowns, popovers
- Fade + scale: `data-[state=open]:animate-in data-[state=open]:fade-in-0 data-[state=open]:zoom-in-95 data-[state=open]:duration-150`
- For dropdowns longer than 250px, also add `data-[state=open]:slide-in-from-top-1`

### 10. Image hovers (galleries, portfolios)
- Wrapping container: `overflow-hidden rounded-xl`
- Image element: `transition-transform duration-500 ease-out group-hover:scale-105`
- Container needs `group` class

### 11. Number/score transitions (already done in URLScanner ring)
For any prominent numeric KPI (Analytics, Dashboard widgets, score displays), add a count-up effect on mount only. Use the existing `useEffect` + `requestAnimationFrame` pattern from `URLScanner.tsx:425-489` — don't introduce a new library.

### 12. Page transitions (subtle, not intrusive)
On the page-level wrapper of every page component, add:
- `animate-in fade-in-0 duration-300`
That's it. No slide, no scale, no stagger at the page level.

### 13. Borders — refined hairlines
Wherever you see `border-slate-200`, `border-slate-300`, or default `border`, prefer:
- Light: `border-black/[0.06]`
- Dark: `dark:border-white/[0.08]`
This matches the existing card primitive. Apply opportunistically — don't refactor every border, but when you touch a component for other polish, fix this too.

### 14. Shadow ramp — consistency
Use only these shadows (matches existing):
- `shadow-sm` → resting cards
- `shadow-md` → hovered cards
- `shadow-lg` → modals, popovers, command palette
- `shadow-xl` → reserved for marketing/hero elements only (don't add new uses)

### 15. Spacing rhythm (apply lightly)
When you see padding inconsistencies on the same surface (e.g., one card has `p-4`, the next `p-5`, the next `p-6`), normalize to the **largest** value used in that surface. Don't normalize across pages.

## Decision log

If you're unsure whether a change qualifies as "polish" vs "refactor":
- ✅ Adding `transition-all duration-200` to a component: polish.
- ✅ Adding `hover:-translate-y-0.5` to a card: polish.
- ✅ Replacing a `<Loader2 />` with a `<Skeleton />` in a 400px loading region: polish.
- ✅ Replacing `border-slate-200` with `border-black/[0.06]`: polish.
- ❌ Renaming a component: refactor.
- ❌ Changing a `useState` to a `useReducer`: refactor.
- ❌ Adding a new prop to a component: feature change.
- ❌ Removing dead code: cleanup, not polish.
- ❌ Changing the order of `<div>`s: layout change.
