How this is checked

The rule

Each heading must satisfy currentLevel − previousLevel ≤ 1. This is the exact check axe-core applies.

What that means
  • Going deeper, levels can only increase by one (h1 → h2, never h1 → h3)
  • Going shallower is unrestricted (h4 → h2 or h4 → h1 is fine)
  • The first heading on the page is always allowed
  • The page must contain at least one h1 (separate rule)
Source

Identical to the heading-order rule in axe-core, the engine used by Lighthouse, Deque, and most accessibility auditors.
